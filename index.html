Subconjuntos SQL

DML, DCL, DDL

• Comandos DML (Data Manipulation Language)


A DML é usada para trabalhar com os dados que já estão dentro de uma tabela no banco de dados. Com ela podemos incluir, mudar ou até apagar informações de forma prática.


  • SELECT: usado para consultar e visualizar os dados salvos.
  • INSERT INTO: serve para adicionar novos dados.
  • UPDATE: usado quando queremos alterar algum dado que já existe.
  • DELETE FROM: remove informações que não são mais necessárias.

• Comandos DDL (Data Definition Language)


O DDL é um subconjunto do SQL usado para definir a estrutura de objetos em um banco de dados, como tabelas, índices e esquemas. Seus principais comandos são:


  • CREATE DATABASE: significa criar um banco de dados sem usar o comando CREATE DATABASE tradicional do SQL.
  • USE: Altera o contexto de banco de dados para o banco de dados ou instantâneo de banco de dados especificado.
  • CREATE TABLE:Usado para criar novos objetos no banco, como tabelas.
  • ALTER TABLE: Modifica um objeto existente no banco, pode adicionar, remover ou alterar informações da tabela.
  • DROP COLUMN: Remove completamente um objeto dentro do banco, incluindo sua estrutura e dados.
  • DESCRIBE: é usado para mostrar a estrutura de uma tabela, como os nomes das colunas, tipos de dados e restrições.

• Cláusulas de filtro


As cláusulas de filtro em SQL são comandos que você usa para selecionar linhas específicas de uma tabela. Imagine que uma tabela é uma planilha gigante. As cláusulas de filtro são o que você usa para dizer ao banco de dados: "Eu só quero ver as linhas que atendem a estas condições".


Where


A cláusula WHERE é a mais fundamental. Ela é usada para filtrar registros com base em uma condição. Pense nela como a sua principal ferramenta de seleção.


Operadores Lógicos (AND, OR, NOT)


  • AND: Usado para combinar duas ou mais condições. A linha só será retornada se TODAS as condições forem verdadeiras.
  • OR: Usado para combinar duas ou mais condições. A linha será retornada se QUALQUER UMA das condições for verdadeira.
  • NOT: Usado para negar uma condição. Retorna as linhas que NÃO satisfazem a condição.

LIKE


A cláusula LIKE é usada para procurar por padrões específicos em uma coluna de texto. Ela é muito útil quando você não sabe o valor exato, mas sabe uma parte dele.


• Consultas e relacionamentos


Uma consulta em SQL (Structured Query Language) é uma instrução usada para interagir com um banco de dados, sendo a SELECT a mais comum. A principal função da cláusula SELECT é recuperar dados de uma ou mais tabelas.


SELECT


A instrução SELECT é a base de todas as consultas. Ela especifica quais colunas você quer ver e de qual tabela. É a sua forma de pedir ao banco de dados para te mostrar informações.


JOIN


A cláusula JOIN é a ferramenta usada para combinar linhas de duas ou mais tabelas com base em uma coluna em comum, ou seja, para unir dados relacionados.
